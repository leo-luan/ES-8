Introdução

  • Modelagem de Sistemas é o processo de desenvolvimento de modelos abstratos de um sistema, em que cada modelo apresenta uma visão ou perspectiva diferente do sistema.
  • Geralmente representa o sistema com alguma notação gráfica, como o UML (Unified Modeling Language).
  • Modelos são usados em praticamente todas as fases do Processo de Software.
  • Por ser abstrato, um modelo deixa de fora detalhes específicos.
  • A partir de diferentes perspectivas, um modelo pode se limitar a representar somente um conjunto de características relevantes a visão em questão.
  • Perspectivas diferentes já foram catalogadas:
    • Perspectiva externa - modela o contexto ou ambiente do sistema.
    • Perspectiva de interação - modela as interações entre o sistema e o ambiente, ou entre seus componentes
    • Perspectiva estrutural - modela a organização dos componentes do sistema
    • Perspectiva comportamental - modela a dinâmica do sistema e as reações aos eventos de usuário
  • Os modelos e perspectivas são importantes para:
    • Facilitar a discussão sobre um novo sistema
    • Documentar um sistema já existente
    • Apoiar a implementação do sistema com descrições visuais detalhadas

Modelos de Contexto

  • Descreve visualmente detalhes contextuais e de ambiente do sistema.
  • Mostra os limites do sistema, as funcionalidades que serão implementadas, os processos manuais e automatizados.
  • Muitas vezes, novos sistemas devem interagir como sistemas legados, ou componentes de outros sistemas. Isso deve ficar claro no modelo de contexto.
  • Geralmente o Diagrama de Atividades da UML é utilizado para modelar o contexto. Esse diagrama é destinado a mostrar as atividades que compõem um processo do sistema.

Modelos de Interação

  • Descreve a comunicação entre o sistema e o ambiente externo (usuário e outros sistemas), e entre seus componentes.
  • Conhecer as características da interação pode prevenir que erros de comunicação aconteçam.
  • Diagramas de Caso de Uso são usados para descrever interações entre o sistema e o mundo externo.
  • Diagramas de sequência são usados para modelar as interações entre os componentes internos do sistema.

Modelos Estruturais

  • Exibem a organização de um sistema em termos de seus componentes e relacionamentos.
  • É muito usado na fase de projeto de Arquitetura de Software.
  • Diagramas de Classes e Diagramas de Componentes podem ser usados para esse tipo de modelagem.

Modelos Comportamentais

  • Descrevem visualmente o comportamento do sistema durante a execução.
  • Mostram o que deve acontecer quando algum estímulo ou evento acontece.
  • Podem ser usados Diagramas de Atividades ou Diagramas de Estado para esse tipo de modelagem.
